babel基础知识

polyfill 和 runtime的区别#

babel 把javascript分为syntax([ˈsɪntæks]:语法；句法)和api，babel只起到转换syntax的作用，那么这个时候就还需要polyfill来处理api部分的内容。

syntax vs api

  • syntax：在javascript运行时无法重写的，可以称之为syntax，比如:let,const,箭头函数，装饰器等等，在不支持的浏览器上面，无论怎么样都不支持let申明，所以只能直接把let改成支持的var

    // 转换前
    let array = [1, 2, 3, 4, 5, 6];
    array.includes(item => item > 2);
    new Promise()
    // 转换后
    var array = [1, 2, 3, 4, 5, 6];
    array.includes(function (item) {
    return item > 2;
    });
    new Promise()
    • 上面示例中：let,箭头函数是无法在运行行重写的，所以属于syntax，Promise，includes是属于api级别的

polyfill#

polyfill 直译的话是垫片的意思，那处理类似assign,includes,map,includes ，某些浏览器 没有的方法 最直接的办法的是 根据 一份浏览器不兼容的表格(这个browserslist已经完成了)，把对应浏览器不支持的语法全部重新写一遍，类似下面这样

if (typeof Object.assign != 'function') {
Object.defineProperty(Object, "assign",·····)
}
if (!Array.prototype.includes){
Object.defineProperty(Array.prototype, 'includes', ·····)
}
if (!Array.prototype.every){
Object.defineProperty(Array.prototype, 'every', ·····)
}

runtime#

runtime主要解决api污染问题，比如：

源码实现polyfill转换runtime转换
const array = [1, 2, 3, 4, 5, 6]
array.includes(item => item > 2)
new Promise()
"use strict";
require("core-js/modules/es.array.includes");
require("core-js/modules/es.object.to-string");
require("core-js/modules/es.promise");
var array = [1, 2, 3, 4, 5, 6];
array.includes(function (item) {
return item > 2;
});
new Promise();
console.log(array);
"use strict";
var _interopRequireDefault = require("@babel/runtime-corejs3/helpers/interopRequireDefault");
var _promise = _interopRequireDefault(require("@babel/runtime-corejs3/core-js-stable/promise"));
var _includes = _interopRequireDefault(require("@babel/runtime-corejs3/core-js-stable/instance/includes"));
var array = [1, 2, 3, 4, 5, 6];
(0, _includes.default)(array).call(array, function (item) {
return item > 2;
});
new _promise.default();

插件执行顺序#

  • plugin会运行在preset之前。
  • plugin从前往后执行。
  • preset刚好相反，从后往前执行。
分析

  • preset实则就是一系列plugin的集合。

  • preset 的逆向顺序主要是为了保证向后兼容，因为大多数用户的编写顺序是 ['es2015', 'stage-0']。这样必须先执行 stage-0 才能确保 babel 不报错。因此我们编排 preset 的时候，也要注意顺序，其实只要按照规范的时间顺序列出即可。

    module.exports = {
    'presets': [
    [
    '@babel/preset-env',
    {
    'useBuiltIns': 'usage',
    'corejs': 3
    },
    '@babel/preset-es2015'
    ]
    ],
    "plugins": [
    [
    "@babel/plugin-transform-runtime",
    {
    "absoluteRuntime": false,
    "corejs": 3,
    "helpers": true,
    "regenerator": true,
    "useESModules": false,
    "version": "7.0.0-beta.0",
    "proposals": true
    }
    ]
    ]
    }

  • 如上配置运行的一个顺序就是：@babel/plugin-transform-runtime > @babel/preset-es2015 > @babel/preset-env

  • 像这种情况项目最终处理api的形式就是以runtime的形式解决，因为@babel/plugin-transform-runtime先运行

    源码实现runtime转换
    const array = [1, 2, 3, 4, 5, 6]
    array.includes(item => item > 2)
    new Promise()
    "use strict";
    var _interopRequireDefault = require("@babel/runtime-corejs3/helpers/interopRequireDefault");
    var _promise = _interopRequireDefault(require("@babel/runtime-corejs3/core-js-stable/promise"));
    var _includes = _interopRequireDefault(require("@babel/runtime-corejs3/core-js-stable/instance/includes"));
    var array = [1, 2, 3, 4, 5, 6];
    (0, _includes.default)(array).call(array, function (item) {
    return item > 2;
    });
    new _promise.default();

babel 配置的环境区分#

可以针对开发环境以及生产环境不同的配置，可实现根据环境变量加载不同的配置。

js 判断环境变量，动态配置#

module.exports = api => {
api.cache(false)
const presets = []
const plugins = []
if (process.env.NODE === 'development') {
preset.push(...)
plugins.push(...)
} else {
preset.push(...)
plugins.push(...)
}
return {
presets,
plugins
}
}

通过env设置不同配置#

module.exports = {
"env": {
"modern": {
"presets": [
"@babel/preset-modules"
]
},
"legacy": {
"presets": [
"@babel/preset-env"
]
}
}
}
